# 思维涌现org —— 个人思想建模与外化系统

> **"知识不是被收集的，而是被消化的；思想不是被记录的，而是被建模的。"**
>
> 这是一套以**领域卡片**为骨架、以**冲突**为动力、以**你的认知档案**为透镜的个人思想建模系统。它帮助你把自己的阅读、思考、创作转化为结构化的知识图，并为未来 agent 的 GraphRAG 提供内容支持。

---

## 核心设计哲学

### 思想是被建模的，不是被堆放的

外来文章和你自己的思考不再是孤立的文档，而是被"消化"进已有卡片的燃料。

- 卡片数量增长缓慢
- 每张卡片内容不断丰富
- 原始文档消化后进入 Archive（可删除）

### 领域卡片 = 骨架 = 目录

领域是你思想中的核心主题空间。所有概念、模型、冲突、实体都在领域内生长。

```
教学领域（顶级领域卡片）
├── 涉及观念：认知脚手架、最近发展区...
├── 涉及原则：认知负荷有限原则...
├── 涉及现象：学生依赖脚手架...
├── 涉及模型：学习螺旋模型...
├── 涉及方法：费曼技巧...
├── 涉及实体：Vygotsky、Bruner...
├── 涉及群体：建构主义学派...
├── 涉及冲突：支持 vs 自主的张力
└── 子领域：课堂互动、课程设计...
```

AI 的工作方式：

1. 读取新内容
2. 扫描所有**顶级领域卡片**
3. 判断内容涉及哪些领域
4. 读取相关领域涉及的实例卡片
5. 进行比较、补充、冲突检测
6. 更新领域卡片和元结构

这就是"渐进式披露"——AI 不需要一次读取全部卡片，而是通过领域卡片树层层深入。

## 工作流

### 你做的事

```
1. 读完一篇文章 / 写完一个想法
2. 放入 00-Inbox/
3. 对 AI 说：
   - 「/construct」← 第一次或需要重建骨架时
   - 「/digest」   ← 日常消化新内容
   - 「/analyze 如何理解 X」← 让 AI 基于你的知识库回答问题
```

### AI 做的事

**`/construct`**：

```
读取种子内容 → 生成领域卡片 → 生成实例卡片 → 设计目录结构
→ 撰写 ontology.md → 生成 Profile 初稿 → 输出建构报告 → 归档种子文档
```

**`/digest`**：

```
读取新内容 → 扫描领域卡片 → 读取相关实例卡片 → 比较与补充
→ 评估 Profile 变化 → 确认后更新/升级 Profile → 调整目录结构
→ 更新领域卡片 → 归档原始文档
```

**`/analyze`**：

```
解析问题 → 读取 Profile → 读取领域与实例卡片 → 组装 Context Package
→ 基于 Context Package 生成回答 → 输出到 04-OutBox/
```

## 卡片的内容标准

### 概念卡片：拒绝空模板

```markdown
---
id: "cognitive-scaffolding"
title: "认知脚手架"
type: "notion"
created: "2026-07-07"
updated: "2026-07-07"
sources:
  - "来自《某书》"
relations:
  - target: "zone-of-proximal-development"
    type: "extends"
tags:
  - "教学"
status: "growing"
---

# 认知脚手架

> 教师为学习者提供的临时性支持结构，帮助其完成当前独立能力尚无法完成的任务。

## 核心内涵

### 两个不可或缺的特征
1. **目标是赋能而非代替**
2. **最终宿命是被拆除**

## 实例（≤ 6 条）

- **来自《某书》**：...
```

## 与旧系统的区别

| 维度 | 旧系统 | 新系统 |
|------|--------|--------|
| 骨架 | 关系卡片 | 领域卡片 |
| Inbox 分类 | articles/ + my-thoughts/ | 统一入口 |
| Profile | 单一文件 | 多文件档案 |
| 目录结构 | 用户预先固定 | LLM 根据内容涌现 |
| 输出 | 无 | 04-OutBox/ |
| 机器可读性 | 较弱 | YAML frontmatter |
| 目标 | 读书笔记 | 个人思想建模 + GraphRAG 支持 |
